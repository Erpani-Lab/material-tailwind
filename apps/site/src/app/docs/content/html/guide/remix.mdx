# Material Tailwind with Remix

Learn how to set up Material Tailwind with Remix. This guide will walk you through installation, configuration, and basic usage.

<br />

## Setting up Remix with Tailwind CSS

Start by creating a new Remix project and installing the required dependencies:

<CodePreviewTailwindClasses codeSnippet={`npx create-remix@latest
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p`} />

<br />

## Configure your template paths

Update the content paths in your `tailwind.config.js` or `tailwind.config.ts`:

<CodePreviewTailwindClasses codeSnippet={`// For JavaScript (tailwind.config.js)
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./app/**/*.{js,jsx,ts,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

// For TypeScript (tailwind.config.ts)
import type { Config } from 'tailwindcss'

export default {
  content: [
    "./app/**/*.{js,jsx,ts,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
} satisfies Config`} />

<br />

## Add the Tailwind directives

Create a new CSS file in `app/styles/tailwind.css` and add the Tailwind directives:

<CodePreviewTailwindClasses codeSnippet={`@tailwind base;
@tailwind components;
@tailwind utilities;`} />

<br />

## Import the styles

Import the styles in your `app/root.tsx`:

<CodePreviewTailwindClasses codeSnippet={`import type { LinksFunction } from "@remix-run/node";
import styles from "~/styles/tailwind.css";

export const links: LinksFunction = () => [
  { rel: "stylesheet", href: styles }
];

export default function App() {
  return (
    <html lang="en">
      <head>
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
        <LiveReload />
      </body>
    </html>
  );
}`} />

<br />

## Install Material Tailwind

Install @material-tailwind/html:

<CodePreviewTailwindClasses codeSnippet={`npm i @material-tailwind/html`} />

<br />

## Example Usage

Here's how to use Material Tailwind components in your Remix routes:

<CodePreviewTailwindClasses codeSnippet={`// app/routes/_index.tsx
import { useEffect } from "react";
import { useHydrated } from "@remix-run/react";
import { initAccordion } from "@material-tailwind/html"; // ESM & CJS

export default function Index() {
  const isHydrated = useHydrated();

  useEffect(() => {
    if (isHydrated) {
      initAccordion();
    }
  }, [isHydrated]);

  return (
    <div className="w-full">
      <div className="group block w-full" data-accordion-container data-accordion-mode="exclusive">
        <button
          data-accordion-toggle
          data-accordion-target="#accordion-1"
          aria-expanded="false"
          className="flex w-full items-center justify-between p-4 text-left"
        >
          <span>What is Material Tailwind?</span>
          <svg
            data-accordion-icon
            className="h-5 w-5 rotate-0 transition-transform"
            fill="none"
            stroke="currentColor"
            viewBox="0 0 24 24"
            xmlns="http://www.w3.org/2000/svg"
          >
            <path
              strokeLinecap="round"
              strokeLinejoin="round"
              strokeWidth="2"
              d="M19 9l-7 7-7-7"
            ></path>
          </svg>
        </button>
        <div
          id="accordion-1"
          className="overflow-hidden transition-all duration-300"
        >
          <div className="p-4">
            <p>
              Material Tailwind is an easy to use components library for Tailwind CSS.
            </p>
          </div>
        </div>
      </div>
    </div>
  );
}`} />

## TypeScript Configuration

For TypeScript projects, ensure your `tsconfig.json` includes:

<CodePreviewTailwindClasses codeSnippet={`{
  "compilerOptions": {
    "moduleResolution": "node",
  },
}`} />

<br />

> **Note**: The TypeScript configuration above includes Remix-specific settings to ensure proper type checking and module resolution when using Material Tailwind components.

## Server-Side Rendering

Since Remix uses server-side rendering, make sure to initialize components after hydration:

<CodePreviewTailwindClasses codeSnippet={`import { useHydrated } from "@remix-run/react";
import { initAccordion } from "@material-tailwind/html";

export default function Component() {
  const isHydrated = useHydrated();

  useEffect(() => {
    if (isHydrated) {
      initAccordion();
    }
  }, [isHydrated]);

  return (
    // Component content
  );
}`} />

<br />

> **Note**: For more examples and detailed documentation of each component, check out the [Accordion](/docs/html/accordion) component.
